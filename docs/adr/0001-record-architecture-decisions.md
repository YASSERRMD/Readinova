# ADR 0001: Record Architecture Decisions

## Status

Accepted

## Context

The platform will evolve across services, data stores, and client surfaces. Decisions that affect architecture, operations, or long-term maintenance need a durable record.

## Decision

Architecture decisions will be captured as numbered ADR files under `docs/adr`.

## Consequences

Each significant architecture choice has a clear rationale and can be reviewed independently from implementation changes.
